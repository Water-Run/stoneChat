# Windows XP 150% 查看窗口设计

## 目标

为现有 VMware Windows XP 虚拟机提供一个可长期使用的独立查看入口：XP 来宾分辨率始终保持 `1024×768`，画面在当前 2× HiDPI 主机上以相对 VMware 当前 1:1 画面约 `150%` 的物理尺寸显示，窗口内不出现大面积黑边，并支持键盘和鼠标操作。

不修改 XP 的 DPI、字体或桌面布局，不保存 XP 登录密码，也不修改 VMware 系统库。

## 方案比较

1. **独立 GTK-VNC 查看窗口（采用）**：复用虚拟机已启用的本地 VNC 端口 `5997` 和系统现有的 GTK-VNC 2.0。可以精确控制显示尺寸，风险低，并能固化为桌面应用入口。
2. **VMware 全屏拉伸**：不改变来宾分辨率，但只能按屏幕尺寸自适应，不能固定为 150%，且 4:3 画面仍会留下侧边黑带。
3. **修改 Workstation 运行时或系统库**：可尝试解除普通窗口的灰色“Stretch Guest”，但升级后容易失效，也可能导致 Workstation 崩溃，不采用。

## 架构与文件

- `~/.local/bin/vmware-xp-150`：Python 3/GTK 3 启动器和查看窗口。
- `~/.local/share/applications/vmware-xp-150.desktop`：应用菜单入口，显示名称为“Windows XP（150%）”。
- 依赖现有组件：`python3`、PyGObject、GTK 3、GTK-VNC 2.0、`vmrun`、`vmcli`。
- 虚拟机路径固定为 `/home/waterrun/VM/Windows/Windows_XP_SP3/Windows XP Professional/Windows XP Professional.vmx`。

不向 stoneChat 产品代码加入运行时依赖；本仓库只保存本设计与实施计划。

## 启动与显示流程

1. 启动器检查目标 VM 是否已运行；未运行时使用 `vmrun start <vmx> nogui` 启动。
2. 等待本机 `127.0.0.1:5997` 可连接，超时则显示中文错误并退出。
3. 通过 GTK-VNC 共享连接到 VM，禁用“客户端请求改变来宾分辨率”。
4. 连接后检查帧缓冲尺寸。若不是 `1024×768`，调用 `vmcli ... MKS SetGuestResolution 1024 768`，等待尺寸恢复后再显示。
5. 根据 GDK 窗口缩放因子计算内容区逻辑尺寸：目标物理画面为 `1536×1152`，在当前 2× 主机缩放下内容区为 `768×576` 逻辑像素。GTK-VNC 将 `1024×768` 等比映射到该区域，因此没有比例不匹配产生的黑边。
6. 查看窗口关闭时仅断开 VNC，不关闭或挂起虚拟机。

窗口标题显示“Windows XP — 1024×768 @ 150%”。画面保持 4:3，缩放使用平滑插值；键盘和鼠标由 GTK-VNC 的标准交互处理。

## 错误处理与安全

- VM 启动失败、VNC 端口超时或 GTK-VNC 连接失败时，显示具体中文错误；不循环重启 VM。
- 分辨率恢复失败时停止显示并报告当前尺寸，避免悄悄使用错误分辨率。
- 单实例运行：重复点击桌面入口时激活已有窗口，不创建多个输入会话。
- XP 登录密码只在需要时由用户在来宾系统内手工输入，不写入脚本、桌面文件、文档或日志。
- 不覆盖 VMware Workstation 原入口；用户仍可按需打开原控制台。

## 验证

1. 几何计算测试覆盖主机缩放因子 1× 和 2×，确认当前环境得到 `768×576` 逻辑内容区、`1536×1152` 物理画面。
2. 启动前后通过 VNC 截图尺寸验证来宾始终为 `1024×768`。
3. 检查 GTK 查看窗口内容区为 4:3，且没有额外黑边。
4. 验证鼠标点击和键盘输入可送达 XP。
5. 关闭并从应用菜单重新打开，确认桌面入口和 150% 设置持久有效。
6. 确认关闭查看窗口后 VM 仍在运行。
